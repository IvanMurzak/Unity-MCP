✨ توسعه‌دهنده بازی با هوش مصنوعی — Unity SKILLS، MCP

`Unity MCP` یک دستیار توسعه بازی مبتنی بر هوش مصنوعی **برای Editor و Runtime** است. **Claude**، **Cursor** و **Windsurf** را از طریق MCP به Unity متصل کنید. گردش‌کارها را خودکار کنید، کد تولید کنید و **هوش مصنوعی را درون بازی‌هایتان فعال کنید**.

برخلاف سایر ابزارها، این پلاگین **درون بازی کامپایل‌شده‌تان** کار می‌کند و امکان دیباگ بلادرنگ با هوش مصنوعی و تعامل بازیکن با AI را فراهم می‌آورد.

# شروع سریع

در سه مرحله راه‌اندازی کنید:

1. **[نصب پلاگین](#step-1-install-unity-mcp-plugin)** — فایل `.unitypackage` نصب‌کننده را دانلود کنید یا دستور `openupm add com.ivanmurzak.unity.mcp` را اجرا کنید
   > **جایگزین:** `npx unity-mcp-cli install-plugin ./MyUnityProject` — به [مستندات CLI](https://github.com/IvanMurzak/Unity-MCP/blob/main/cli/README.md) مراجعه کنید
2. **[انتخاب یک عامل هوش مصنوعی](#step-2-install-ai-agent)** — Claude Code، Claude Desktop، GitHub Copilot، Cursor یا هر گزینه دیگری
3. **[پیکربندی عامل هوش مصنوعی](#step-3-configure-ai-agent)** — در Unity مسیر `Window/AI Game Developer` را باز کرده و روی **Auto-generate skills** (پیشنهادی) یا **Configure MCP** کلیک کنید
   > **جایگزین:** `npx unity-mcp-cli setup-skills claude-code ./MyUnityProject` — به [مستندات CLI](https://github.com/IvanMurzak/Unity-MCP/blob/main/cli/README.md) مراجعه کنید

همین. از هوش مصنوعی بخواهید *«۳ مکعب در یک دایره با شعاع ۲ بساز»* و شاهد اجرای آن باشید. ✨

> - ❌ `C:/My Projects/My Project`
> - ❌ `C:/MyProjects/My Project`

### گزینه ۱ - نصب‌کننده

- **[⬇️ دانلود نصب‌کننده](https://github.com/IvanMurzak/Unity-MCP/releases/latest/download/AI-Game-Dev-Installer.unitypackage)**
- **📂 فایل نصب‌کننده را در پروژه Unity وارد کنید**
  > - می‌توانید روی فایل دوبار کلیک کنید - Unity به طور خودکار آن را باز می‌کند
  > - یا: ابتدا Unity Editor را باز کنید، سپس روی `Assets/Import Package/Custom Package` کلیک کرده و فایل را انتخاب کنید

### گزینه ۲ - CLI (پیشنهادی)

پلاگین را از طریق [`unity-mcp-cli`](https://github.com/IvanMurzak/Unity-MCP/blob/main/cli/README.md) نصب کنید — بدون نیاز به Unity Editor:

```bash
# ۱.۱ نصب unity-mcp-cli
npm install -g unity-mcp-cli

# ۱.۲ (اختیاری) نصب Unity
unity-mcp-cli install-unity

# ۱.۳ (اختیاری) ایجاد پروژه Unity
unity-mcp-cli create-project ./MyUnityProject

# ۲. نصب «AI Game Developer» در پروژه Unity
unity-mcp-cli install-plugin ./MyUnityProject

# ۳. ورود به سرور ابری
unity-mcp-cli login ./MyUnityProject

# ۴. باز کردن پروژه Unity (اتصال خودکار و تولید مهارت‌ها)
unity-mcp-cli open ./MyUnityProject

# ۵. منتظر آماده شدن Unity Editor بمانید
unity-mcp-cli wait-for-ready ./MyUnityProject
```

> برای تمام دستورات موجود، [مستندات کامل CLI](https://github.com/IvanMurzak/Unity-MCP/blob/main/cli/README.md) را ببینید.

## غیرفعال کردن اعلان‌های به‌روزرسانی برای کل تیم

این پلاگین هنگام راه‌اندازی Editor، یک پاپ‌آپ به‌روزرسانی نمایش می‌دهد اگر نسخه جدیدتری در OpenUPM موجود باشد. به‌طور پیش‌فرض، هر عضو تیم این پاپ‌آپ را می‌بیند تا زمانی که روی *«دیگر نشان نده»* کلیک کند (که یک تنظیم per-user است و روی دستگاه آن‌ها ذخیره می‌شود).

برای پروژه‌های Unity با چند نفر که یک مهندس مسئول نسخه‌بندی پلاگین است، می‌توانید پاپ‌آپ را برای **کل تیم** غیرفعال کنید؛ کافی است **Edit ▸ Project Settings ▸ AI Game Developer** را باز کرده و گزینه *«Disable update notifications for the entire team»* را فعال کنید. این تنظیم در `ProjectSettings/AI-Game-Developer-UpdateSettings.asset` ذخیره می‌شود و فقط یک بار در هر پروژه باید تنظیم شود — آن فایل را commit کنید تا هر عضو تیمی که آن را pull کند، پاپ‌آپ برایش سرکوب شود.

همین گزینه از طریق **Tools ▸ AI Game Developer ▸ Updates ▸ Disable Update Notifications (Team)** در نوار منو نیز قابل دسترسی است.

## ویژگی‌های پیشرفته برای LLM

Unity MCP ابزارهای پیشرفته‌ای ارائه می‌دهد که LLM را قادر می‌سازد سریع‌تر و مؤثرتر کار کند، از اشتباهات جلوگیری کرده و هنگام بروز خطا خودش را اصلاح کند. همه چیز برای رسیدن به اهداف شما به صورت کارآمد طراحی شده است.

### قابلیت‌های اصلی

- ✔️ **ابزارهای آماده برای عامل** - هر چیزی که نیاز دارید را در ۱ تا ۲ مرحله پیدا کنید
- ✔️ **کامپایل فوری** - کامپایل و اجرای کد #C با استفاده از `Roslyn` برای تکرار سریع‌تر
- ✔️ **دسترسی کامل به asset** - دسترسی خواندن/نوشتن به assetها و اسکریپت‌های #C
- ✔️ **بازخورد هوشمند** - بازخورد مثبت و منفی خوب‌توصیف‌شده برای درک صحیح مشکلات

### ویژگی‌های مبتنی بر Reflection

- ✔️ **ارجاعات به اشیاء** - ارائه ارجاع به اشیاء موجود برای کد #C فوری
- ✔️ **دسترسی به داده‌های پروژه** - دسترسی کامل به تمام داده‌های پروژه در قالبی قابل خواندن
- ✔️ **تغییرات دانه‌ریز** - پر کردن و ویرایش هر قطعه‌ای از داده‌های پروژه
- ✔️ **کشف متد** - یافتن هر متدی در کل پایگاه کد، از جمله فایل‌های DLL کامپایل‌شده
- ✔️ **اجرای متد** - فراخوانی هر متدی در کل پایگاه کد
- ✔️ **پارامترهای پیشرفته** - ارائه هر property برای فراخوانی متد، حتی ارجاع به اشیاء موجود در حافظه
- ✔️ **Unity API زنده** - Unity API بلافاصله در دسترس است — حتی وقتی Unity تغییر می‌کند، API تازه دریافت می‌کنید
- ✔️ **خود-مستندساز** - دسترسی به توضیحات قابل خواندن توسط انسان از هر `class`، `method` یا `property` از طریق ویژگی‌های `Description`

# سفارشی‌سازی ابزارها

**[Unity MCP](https://github.com/IvanMurzak/Unity-MCP)** از توسعه `MCP Tool`، `MCP Resource` و `MCP Prompt` سفارشی توسط صاحبان پروژه پشتیبانی می‌کند. سرور MCP داده‌ها را از `Unity MCP Plugin` دریافت کرده و آن‌ها را به یک client ارائه می‌دهد. هر کسی در زنجیره ارتباطی MCP اطلاعات مربوط به ویژگی‌های جدید MCP را دریافت می‌کند، که ممکن است LLM در جایی تصمیم بگیرد از آن‌ها استفاده کند.

## افزودن `Tool` سفارشی

برای افزودن یک `Tool` سفارشی، نیاز دارید:

1. یک کلاس با ویژگی `AiToolType`
2. یک متد در کلاس با ویژگی `AiTool`
3. *اختیاری:* افزودن ویژگی `Description` به هر آرگومان متد برای کمک به فهم LLM
4. *اختیاری:* استفاده از `string? optional = null` با `?` و مقادیر پیش‌فرض برای علامت‌گذاری پارامترها به عنوان `optional` برای LLM

> توجه کنید که خط `MainThread.Instance.Run(() =>` به شما امکان می‌دهد کد را در thread اصلی اجرا کنید، که برای تعامل با Unity API الزامی است. اگر به این نیاز ندارید و اجرای ابزار در thread پس‌زمینه قابل قبول است، برای بهره‌وری بیشتر از thread اصلی استفاده نکنید.

## افزودن `MCP Prompt` سفارشی

`MCP Prompt` به شما امکان می‌دهد promptهای سفارشی را به مکالمه با LLM تزریق کنید. از دو نقش فرستنده پشتیبانی می‌کند: User و Assistant. این یک راه سریع برای دستورالعمل‌دادن به LLM جهت انجام وظایف خاص است. می‌توانید promptها را با استفاده از داده‌های سفارشی تولید کنید، لیست‌ها یا هر اطلاعات مرتبط دیگری ارائه دهید.

# استفاده از Runtime (درون بازی)

از **[Unity MCP](https://github.com/IvanMurzak/Unity-MCP)** در بازی/اپلیکیشن خود استفاده کنید. از Tools، Resources یا Prompts استفاده کنید. به‌طور پیش‌فرض هیچ ابزاری وجود ندارد و باید ابزارهای سفارشی خود را پیاده‌سازی کنید.

## نمونه: ربات بازی شطرنج مبتنی بر هوش مصنوعی

یک بازی شطرنج کلاسیک وجود دارد. بیایید منطق ربات را به LLM برون‌سپاری کنیم. ربات باید بر اساس قوانین بازی حرکت خود را انجام دهد.

## چرا استفاده از Runtime ضروری است؟

موارد استفاده بسیاری وجود دارد؛ تصور کنید روی یک بازی شطرنج با ربات کار می‌کنید. می‌توانید تصمیم‌گیری ربات را با نوشتن چند خط کد به LLM برون‌سپاری کنید.

# راه‌اندازی `MCP Server` در Unity

سرور **[Unity MCP](https://github.com/IvanMurzak/Unity-MCP)** از گزینه‌های راه‌اندازی متعدد و استقرار Docker پشتیبانی می‌کند. هر دو پروتکل انتقال پشتیبانی می‌شوند: `streamableHttp` و `stdio`. اگر نیاز به سفارشی‌سازی یا استقرار Unity MCP Server در ابر دارید، این بخش برای شماست. [بیشتر بخوانید...](https://github.com/IvanMurzak/Unity-MCP/blob/main/docs/mcp-server.md)

## متغیرها

صرف نظر از گزینه راه‌اندازی انتخابی، همه از پیکربندی سفارشی با استفاده از Environment Variables و Command Line Arguments پشتیبانی می‌کنند. با مقادیر پیش‌فرض کار می‌کند؛ اگر فقط می‌خواهید آن را راه‌اندازی کنید، وقت خود را برای متغیرها هدر ندهید. فقط مطمئن شوید که Unity Plugin هم مقادیر پیش‌فرض دارد، به خصوص `--port`، که باید برابر باشند.

| متغیر محیطی | آرگومان خط فرمان | توضیح |
| --- | --- | --- |
| `MCP_PLUGIN_PORT` | `--port` | پورت اتصال **Client** -> **Server** <- **Plugin** (پیش‌فرض: ۸۰۸۰) |
| `MCP_PLUGIN_CLIENT_TIMEOUT` | `--plugin-timeout` | تایم‌اوت اتصال **Plugin** -> **Server** (میلی‌ثانیه) (پیش‌فرض: ۱۰۰۰۰) |
| `MCP_PLUGIN_CLIENT_TRANSPORT` | `--client-transport` | نوع انتقال **Client** -> **Server**: `stdio` یا `streamableHttp` (پیش‌فرض: `streamableHttp`) |
| `MCP_AUTHORIZATION` | `--authorization` | حالت احراز هویت برای اتصالات **Client** ورودی: `none` یا `required` (پیش‌فرض: `none`) |
| `MCP_PLUGIN_TOKEN` | `--token` | توکن Bearer مورد نیاز از **Client** هنگامی که `--authorization=required` است (پیش‌فرض: تنظیم نشده) |
| `MCP_PLUGIN_IDLE_TIMEOUT_SECONDS` | `--idle-timeout-seconds` | خاموش کردن سرور پس از این تعداد ثانیه بدون اتصال (پیش‌فرض: ۶۰۰) |

> آرگومان‌های خط فرمان از پیشوند `-` تکی (`-port`) و گزینه بدون پیشوند (`port`) هم پشتیبانی می‌کنند.

> **انتخاب transport:** از `stdio` استفاده کنید وقتی MCP client مستقیماً باینری سرور را راه‌اندازی می‌کند (استفاده محلی — این رایج‌ترین تنظیم است). از `streamableHttp` استفاده کنید وقتی سرور را به عنوان یک process مستقل یا در Docker/ابر اجرا می‌کنید و از طریق HTTP متصل می‌شوید.

## متغیرهای پلاگین

Unity MCP Plugin متغیرهای محیطی (و آرگومان‌های خط فرمان) زیر را هنگام راه‌اندازی می‌خواند تا مقادیر فایل پیکربندی ذخیره‌شده را بازنویسی کند. بازنویسی‌ها در زمان اجرا اعمال می‌شوند؛ در اولین اجرا یا زمانی که یک توکن احراز هویت جدید تولید می‌شود، مقادیر بازنویسی‌شده **در فایل پیکربندی نوشته می‌شوند**. در اجراهای بعدی، بازنویسی‌ها در حافظه اعمال می‌شوند اما به‌طور خودکار ذخیره نمی‌شوند. استثنا `UNITY_MCP_TOOLS` است که از `[JsonIgnore]` استفاده می‌کند و **هرگز ذخیره نمی‌شود** — فقط runtime است.

> آرگومان‌های خط فرمان بر متغیرهای محیطی اولویت دارند. هر دو مقدار فایل پیکربندی ذخیره‌شده را بازنویسی می‌کنند. پرچم‌های کوتاه `-url`، `-token` و `-auth` نام مستعار هستند؛ نام‌های کامل آرگومان `-UNITY_MCP_*` هم قابل قبول هستند.

## Docker 📦

مطمئن شوید Docker نصب شده است. و لطفاً اطمینان حاصل کنید که اگر روی ویندوز هستید، Docker Desktop راه‌اندازی شده باشد.

[راهنمای پیکربندی پیشرفته Docker را بخوانید](https://github.com/IvanMurzak/Unity-MCP/blob/main/docs/DOCKER_DEPLOYMENT.md).

## فایل اجرایی باینری

می‌توانید Unity `MCP Server` را مستقیماً از یک فایل باینری راه‌اندازی کنید. به یک باینری کامپایل‌شده خاص برای معماری CPU شما نیاز خواهید داشت. [صفحه Release GitHub](https://github.com/IvanMurzak/Unity-MCP/releases) را بررسی کنید؛ این صفحه شامل باینری‌های پیش‌کامپایل‌شده برای تمام معماری‌های CPU است.

# نحوه کارکرد معماری Unity MCP

**[Unity MCP](https://github.com/IvanMurzak/Unity-MCP)** به عنوان یک پل بین LLMها و Unity عمل می‌کند. ابزارهای Unity را برای LLM نمایش داده و توضیح می‌دهد، که LLM سپس رابط را درک کرده و ابزارها را طبق درخواست‌های کاربر به‌کار می‌گیرد.

**[Unity MCP](https://github.com/IvanMurzak/Unity-MCP)** را از طریق پنجره `AI Connector` یکپارچه به کلاینت‌های LLM مانند [Claude](https://claude.ai/download) یا [Cursor](https://www.cursor.com/) متصل کنید. کلاینت‌های سفارشی هم پشتیبانی می‌شوند.

این سیستم بسیار قابل توسعه است — می‌توانید `MCP Tools`، `MCP Resource` یا `MCP Prompt` سفارشی را مستقیماً در پایگاه کد پروژه Unity خود تعریف کنید و قابلیت‌های جدیدی را برای کلاینت‌های هوش مصنوعی یا اتوماسیون نمایش دهید. این ویژگی Unity MCP را به یک پایه انعطاف‌پذیر برای ساخت گردش‌کارهای پیشرفته، نمونه‌سازی سریع و یکپارچه‌سازی ویژگی‌های مبتنی بر هوش مصنوعی در فرآیند توسعه‌تان تبدیل می‌کند.

## MCP چیست؟

MCP مخفف Model Context Protocol است. به طور خلاصه، `USB Type-C` دنیای هوش مصنوعی، به ویژه برای LLM (مدل زبانی بزرگ) است. به LLM می‌آموزد چگونه از ویژگی‌های خارجی استفاده کند؛ مانند Unity Engine در این مورد، یا حتی متد #C سفارشی در کد شما. [مستندات رسمی](https://modelcontextprotocol.io/).

## `عامل هوش مصنوعی` چیست؟

یک اپلیکیشن با پنجره چت است. ممکن است عامل‌های هوشمندی برای عملکرد بهتر داشته باشد، ابزارهای پیشرفته MCP یکپارچه‌شده داشته باشد. به طور کلی، یک MCP Client خوب ۵۰ درصد موفقیت هوش مصنوعی در اجرای یک وظیفه است. به همین دلیل انتخاب بهترین گزینه بسیار مهم است.

## `MCP Server` چیست؟

یک پل بین `MCP Client` و «چیز دیگری» است؛ در این مورد خاص، Unity Engine است. این پروژه شامل `MCP Server` می‌شود.

## `MCP Tool` چیست؟

`MCP Tool` یک تابع یا متد است که LLM می‌تواند برای تعامل با Unity فراخوانی کند. این ابزارها به عنوان پل بین درخواست‌های زبان طبیعی و عملیات واقعی Unity عمل می‌کنند. وقتی از هوش مصنوعی می‌خواهید «یک مکعب بساز» یا «رنگ متریال را تغییر بده»، از MCP Tools برای اجرای این اقدامات استفاده می‌کند.

**ویژگی‌های کلیدی:**

- **توابع قابل اجرا** که عملیات خاصی را انجام می‌دهند
- **پارامترهای نوع‌دار** با توضیحاتی که به LLM کمک می‌کند بفهمد چه داده‌ای ارائه دهد
- **مقادیر برگشتی** که بازخورد موفقیت یا شکست عملیات را می‌دهند
- **آگاه از thread** - می‌توانند روی thread اصلی برای فراخوانی Unity API یا thread پس‌زمینه برای پردازش سنگین اجرا شوند

### چه زمانی از `Tool` استفاده کنیم

- **خودکارسازی وظایف تکراری** - ابزارهایی برای عملیاتی که مکرراً انجام می‌دهید بسازید
- **عملیات پیچیده** - چندین فراخوانی Unity API را در یک ابزار آسان‌استفاده ترکیب کنید
- **گردش‌کارهای خاص پروژه** - ابزارهایی بسازید که ساختار و قراردادهای خاص پروژه‌تان را درک می‌کنند
- **وظایف مستعد خطا** - ابزارهایی بسازید که شامل اعتبارسنجی و مدیریت خطا هستند
- **منطق بازی سفارشی** - سیستم‌های بازی خود را برای ایجاد محتوای پویا به هوش مصنوعی نمایش دهید

**مثال‌ها:**

- ایجاد و پیکربندی GameObjectها با کامپوننت‌های خاص
- پردازش دسته‌ای assetها (تکسچرها، متریال‌ها، prefabها)
- راه‌اندازی نورپردازی و افکت‌های post-processing
- تولید هندسه سطح یا قرار دادن اشیاء به صورت procedural
- پیکربندی تنظیمات فیزیک یا لایه‌های collision

## `MCP Resource` چیست؟

`MCP Resource` دسترسی فقط‌خواندنی به داده‌ها درون پروژه Unity شما فراهم می‌کند. برخلاف MCP Tools که اقداماتی انجام می‌دهند، Resources به LLM امکان می‌دهد وضعیت فعلی، assetها و پیکربندی پروژه شما را بررسی کرده و درک کند. آن‌ها را به عنوان «حسگرهایی» در نظر بگیرید که به هوش مصنوعی زمینه‌ای درباره پروژه‌تان می‌دهند.

**ویژگی‌های کلیدی:**

- **دسترسی فقط‌خواندنی** به داده‌های پروژه و اشیاء Unity
- **اطلاعات ساختاریافته** ارائه‌شده در قالبی که LLM می‌تواند درک کند
- **داده‌های بلادرنگ** که وضعیت فعلی پروژه شما را منعکس می‌کنند
- **آگاهی زمینه‌ای** که به هوش مصنوعی کمک می‌کند تصمیمات آگاهانه‌تری بگیرد

### چه زمانی از `MCP Resource` استفاده کنیم

- **تحلیل پروژه** - اجازه دهید هوش مصنوعی ساختار، assetها و سازمان پروژه شما را درک کند
- **کمک در دیباگ** - اطلاعات وضعیت فعلی را برای عیب‌یابی فراهم کنید
- **پیشنهادهای هوشمند** - به هوش مصنوعی زمینه بدهید تا توصیه‌های بهتری ارائه دهد
- **تولید مستندات** - بر اساس وضعیت پروژه به‌طور خودکار مستندات ایجاد کنید
- **مدیریت asset** - به هوش مصنوعی کمک کنید بفهمد چه assetهایی موجود است و ویژگی‌هایشان چیست

**مثال‌ها:**

- نمایش سلسله‌مراتب صحنه و ویژگی‌های GameObject
- فهرست کردن متریال‌ها، تکسچرها و تنظیمات آن‌ها
- نمایش وابستگی‌های اسکریپت و روابط کامپوننت
- نمایش تنظیمات نورپردازی فعلی و پیکربندی render pipeline
- ارائه اطلاعات درباره منابع صوتی، انیمیشن‌ها و سیستم‌های ذرات

## `MCP Prompt` چیست؟

`MCP Prompt` به شما امکان می‌دهد promptهای از پیش تعریف‌شده را به مکالمه با LLM تزریق کنید. این‌ها قالب‌های هوشمندی هستند که می‌توانند زمینه، دستورالعمل‌ها یا دانش را برای هدایت رفتار هوش مصنوعی فراهم کنند. Promptها می‌توانند متن ثابت یا به‌صورت پویا بر اساس وضعیت فعلی پروژه‌تان تولید شوند.

**ویژگی‌های کلیدی:**

- **راهنمای زمینه‌ای** که بر نحوه پاسخ‌دهی هوش مصنوعی تأثیر می‌گذارد
- **نقش‌محور** - می‌تواند شخصیت‌های مختلف را شبیه‌سازی کند (درخواست‌های User یا دانش Assistant)
- **محتوای پویا** - می‌تواند شامل داده‌های بلادرنگ پروژه باشد
- **قالب‌های قابل استفاده مجدد** برای سناریوها و گردش‌کارهای رایج

### چه زمانی از `MCP Prompt` استفاده کنیم

- **ارائه دانش دامنه** - بهترین روش‌ها و استانداردهای کدنویسی خاص پروژه‌تان را به اشتراک بگذارید
- **تنظیم قراردادهای کدنویسی** - قراردادهای نام‌گذاری، الگوهای معماری و سبک کد را مشخص کنید
- **ارائه زمینه درباره ساختار پروژه** - توضیح دهید پروژه چگونه سازماندهی شده و چرا
- **اشتراک‌گذاری دستورالعمل‌های گردش‌کار** - رویه‌های گام‌به‌گام برای وظایف رایج فراهم کنید
- **تزریق دانش تخصصی** - اطلاعاتی درباره ویژگی‌های خاص Unity، assetهای شخص ثالث یا سیستم‌های سفارشی اضافه کنید

**مثال‌ها:**

- «همیشه برای متدهای public از PascalCase و برای فیلدهای private از camelCase استفاده کن»
- «این پروژه از یک سیستم رویداد سفارشی در Scripts/Events/ استفاده می‌کند»
- «هنگام ایجاد عناصر UI، همیشه آن‌ها را به Canvas در Scene/UI/MainCanvas اضافه کن»
- «عملکرد حیاتی است — برای اشیاء با instantiate مکرر، object pooling را ترجیح بده»
- «این پروژه از اصول SOLID پیروی می‌کند — هر تصمیم معماری را توضیح بده»

# مشارکت 💙💛

مشارکت‌ها بسیار ارزشمند هستند. ایده‌هایتان را بیاورید و بیایید توسعه بازی را ساده‌تر از همیشه کنیم! ایده‌ای برای `Tool` یا ویژگی جدید دارید، یا باگی پیدا کرده‌اید و می‌دانید چطور آن را رفع کنید؟

**لطفاً اگر این پروژه را مفید می‌یابید به آن ستاره بدهید 🌟**

1. 👉 [مستندات توسعه را بخوانید](https://github.com/IvanMurzak/Unity-MCP/blob/main/docs/dev/Development.md)
2. 👉 [پروژه را Fork کنید](https://github.com/IvanMurzak/Unity-MCP/fork)
3. Fork را clone کرده و پوشه `./Unity-MCP-Plugin` را در Unity باز کنید
4. قابلیت‌های جدید را در پروژه پیاده‌سازی کنید، commit کرده و به GitHub push کنید
5. Pull Request برای مخزن اصلی [Unity-MCP](https://github.com/IvanMurzak/Unity-MCP/compare)، شاخه `main` ایجاد کنید.
